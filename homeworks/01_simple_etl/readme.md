# Домашнее задание №1

Дублирую описание ДЗ со слайдов:
Soft deadline – 20.03.2026 (+5 баллов)
Hard deadline – 05.06.2026

Нужно реализовать ETL процесс. Функциональные требования:
1. Инкрементальная загрузка. Необходимо загружать данные только за определенный день. Если файла за дату нет, то DAG завершается успешно, ничего не сделав. При повторном запуске должен вести себя корректно (как именно корректно – решать вам) (2 балла)
2. Два слоя хранения. Сырые данных хранятся в папке raw, обработанные в processed (2 балла)
3. Идемпотентность. Повторные запуски/перезапуски или запуски задним числом должны работать корректно (2 балла)
4. Проверка качества данных:
  а) Не пустая таблица на выходе
  б) Нет строк с transaction_id IS NULL
  в) Нет строк с отрицательным значением amount
  г) transaction_id является уникальным (2 балла)
5.Учесть, что DAG может падать. Если падает, то перезапускать. Если умирает, то шлет письмо на почту. Разобраться с max_active_runs и concurrency (1 балл)
6. Очистка устаревших данных. Если данным (сырым) больше 2 недель, их надо удалить (1 балл)
Итого: 10 баллов (15 в soft deadline)
